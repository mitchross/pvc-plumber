> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the three operator-code findings from the 2026-06-09 independent review — the schedule-drift blind spot, the false `manual`-tier semantics, and silently-inert v5 annotations — without expanding v4's permissive write contract.

**Architecture:** All changes stay inside the existing read→classify→plan→execute pipeline. Task 1 makes `observeCurrent` capture `spec.trigger.schedule` so the planner's *already-implemented and already-tested* schedule-drift comparison (`shapeMatches`, `TestPlanFor_EnabledManage_OperatorOwnedScheduleDrifts_WouldUpdate`) goes live. Tasks 3–4 add planner Notes so `/audit` surfaces defaulted tiers and recognized-but-unenforced annotations. No new write paths, no new GVKs, no admission work (v5 stays design-only).

**Tech Stack:** Go, controller-runtime fake client tests (`internal/controller/v4_reconciler_test.go` fixture), pure-function planner/builder tests. `make test`, `make vet`, `make lint`.

**Out of scope (do not do):** v5 strict/admission/source-gating, retention-per-tier changes, RD `restoreAsOf` support, any change to the talos repo other than the final image-digest bump in Task 6 (which is gated on explicit operator go-ahead).

**Review findings this implements:** see talos repo Mink note `pvc-plumber v4.0.1 code review (2026-06-09)`. Cluster exposure at time of writing: 18 daily + 6 hourly managed PVCs; no `manual`/unspecified tiers in use — Task 2/3 close latent traps, Task 1 closes a live gap.

### Task 2: `manual` tier emits a VolSync manual trigger, not a daily cron

`ScheduleFor` maps `TierManual` to a daily cron and `BuildRS` sets it unconditionally, so "manual" actually means "daily at 02:xx". The schedule.go comment claims "the planner decides whether to actually run an RS for manual tier" — it does not. **Locked decision:** `manual` renders `spec.trigger.manual: "backup-on-demand"` (no cron). VolSync fires one sync when the string first appears, then only when the string changes — a human triggers a backup with `kubectl patch replicationsource <pvc> -n <ns> --type merge -p '{"spec":{"trigger":{"manual":"backup-<date>"}}}'`. A human-bumped string is NOT drift (shapeMatches only inspects cron schedules), but note: any *other* drift that triggers `updateOps` rewrites the RS wholesale and resets the string to `backup-on-demand` — acceptable for a homelab manual tier; documented in audit-api.md in Step 8. An RS left with a leftover cron when the tier is `manual` IS drift and gets repaired.

// inertAnnotationNotes discloses annotations the parser recognizes but
// the v4 permissive reconciler does not enforce (their consumers —
// sourcegate, decision engine, admission webhooks — only wire under
// enforce/strict, which validateMode rejects in v4). Without these
// notes a user setting min-backup-age believes protection exists when
// it does not (2026-06-09 review).
func inertAnnotationNotes(in Inputs) []string {
	const suffix = " is recognized but not enforced in v4 permissive mode (v5 design-only)"
	var notes []string
	if in.Spec.MinBackupAgeSet {
		notes = append(notes, labels.AnnotationMinBackupAge+suffix)
	}
	if in.Spec.SkipRestore {
		notes = append(notes, labels.AnnotationSkipRestore+suffix)
	}
	if in.Spec.Mode != "" {
		notes = append(notes, labels.AnnotationMode+suffix)
	}
	if in.Spec.RestoreMode != "" {
		notes = append(notes, labels.AnnotationRestoreMode+suffix)
	}
	return notes
}
